üîç How Prefire Works

1

Parse SourceFiles

Scans all Swift files, detecting #Preview blocks and PreviewProviders with modifiers like .prefireEnabled() or .prefireIgnored().

2

Cache Models

Fingerprints AST & preview bodies (SHA‚Äë256 + timestamps) to skip work on unchanged files.

3

Generate Snapshot Tests

Uses Stencil templates & your .prefire.yml to emit Swift tests executed via PrefireTestsPlugin.

4

Build Playbook Models

Creates PreviewModels.generated.swift grouping previews by UserStory and State for the interactive Playbook view.

üß† Internal Architecture

  • PrefireCore ‚Äî AST parsing, preview detection & caching layer
  • PrefireGenerator ‚Äî Stencil‚Äëbased code generator for tests & models
  • PrefireCacheManager ‚Äî Unifies fingerprint & file‚Äëcache logic
  • PrefireTestsPlugin / PrefirePlaybookPlugin ‚Äî Xcode & SPM integrations
  • prefire CLI ‚Äî Entry point for manual tests / playbook commands

üß† Usage Examples
